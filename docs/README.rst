MDtoPDF Documentation
====================

This directory contains the documentation for the MDtoPDF project. The documentation is written in reStructuredText (RST) format and built using Sphinx.

Documentation Structure
---------------------

The documentation is organized as follows:

* ``source/`` - Contains the source reStructuredText files
* ``source/architecture/`` - Documentation about project architecture and design decisions
* ``source/usage/`` - User guides and usage examples
* ``source/development/`` - Development guidelines and contributing information
* ``source/api/`` - Auto-generated API documentation (created by sphinx-apidoc)
* ``build/`` - Contains the built documentation (generated by sphinx-build)

Building the Documentation
------------------------

To build the documentation, you have two options:

Option 1: Using the build script
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Run the provided build script:

.. code-block:: bash

   # On Unix/Linux/macOS
   ./build_docs.py

   # On Windows
   python build_docs.py

This script will:

1. Generate API documentation using sphinx-apidoc
2. Build the HTML documentation using sphinx-build

Option 2: Manual build
~~~~~~~~~~~~~~~~~~~~~

If you prefer to build the documentation manually:

1. Generate API documentation:

   .. code-block:: bash

      sphinx-apidoc -f -e -M -o source/api ../src/mdtopdf

2. Build the HTML documentation:

   .. code-block:: bash

      sphinx-build -b html source build/html

Viewing the Documentation
-----------------------

After building the documentation, you can view it by opening ``build/html/index.html`` in your web browser.

Contributing to the Documentation
-------------------------------

1. **Adding new pages**: Create new .rst files in the appropriate directory and add them to the toctree in the corresponding index.rst

2. **Updating existing pages**: Edit the .rst files in the source directory

3. **API documentation**: Ensure that docstrings in the code follow Google style format for optimal autodoc output

4. **Build and check**: Always build the documentation and check the output before committing changes

Writing reStructuredText
----------------------

The documentation is written in reStructuredText. Here are some tips:

* Use ``=`` for page titles (overline and underline)
* Use ``-`` for section titles
* Use ``~`` for subsection titles
* Use ``^`` for subsubsection titles
* Indent properly for lists and code blocks
* Use double backticks for inline code: ````code````
* Use code-block directive for code blocks:

  .. code-block:: rst

     .. code-block:: python

        def example():
            """Docstring."""
            return True

For more details, see the `reStructuredText Primer <https://www.sphinx-doc.org/en/master/usage/restructuredtext/basics.html>`_.

Using Autodoc
-----------

The API documentation is generated automatically from docstrings in the code using Sphinx's autodoc extension.

For more information on using autodoc, see ``source/development/autodoc_guide.rst``.

Installing Documentation Dependencies
----------------------------------

To work with the documentation, you need Sphinx and the Read the Docs theme:

.. code-block:: bash

   pip install sphinx sphinx-rtd-theme
