====================
GUI Screenshots Guide
====================

This directory contains screenshots of the MDtoPDF GUI for documentation purposes.

Required Screenshots
------------------

The following screenshots should be maintained and updated whenever the GUI changes:

1. **mdtopdf_gui_screenshot.png**
   - Main application window showing the default tab (Markdown to PDF)
   - Resolution: 800x600 or higher
   - Format: PNG with transparency where applicable

2. **gui_markdown_to_pdf_tab.png**
   - Close-up of the Markdown to PDF conversion tab
   - Should show input and output fields and the convert button

3. **gui_pdf_to_markdown_tab.png**
   - Close-up of the PDF to Markdown conversion tab
   - Should show input and output fields and the convert button

4. **gui_markdown_to_html_tab.png**
   - Close-up of the Markdown to HTML conversion tab
   - Should show input and output fields and the convert button

Creating Screenshots
------------------

Follow these guidelines when creating screenshots:

1. Use a clean, professional desktop background
2. Set the GUI theme to the default (DarkBlue3)
3. Resize the window to an appropriate size (not too large or small)
4. Include sample data in input fields for context
5. Capture the entire window including title bar
6. Use PNG format for best quality

Screenshot Placement in Documentation
-----------------------------------

Reference screenshots in documentation files with the following syntax:

.. code-block:: rst

   .. image:: /_static/images/mdtopdf_gui_screenshot.png
      :alt: MDtoPDF GUI Main Window
      :width: 600px

Ensure the alt text provides adequate description for accessibility purposes.

Updating Screenshots
------------------

Screenshots should be updated when:

1. The GUI layout changes
2. New features are added
3. The styling or theme changes significantly
4. A new version is released with visual changes

When updating screenshots, try to match the previous angle and content where possible
for consistency across documentation versions.
